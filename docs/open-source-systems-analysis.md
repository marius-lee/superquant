# 10个开源攻击性系统分析 — 2026-06-24

## 可以直接借鉴的精华

### 1. daban (Go)
**架构**: Go语言, 高性能执行
**核心参数** (来自真实实战):
- raTh=8 (最低触发涨幅阈值)
- raRate=0.125 (观测期秒均涨幅)
- tnRate=0.005 (观测期秒均换手)
- stockAmt=20 (观测期秒均成交额万元)
- holdCnt=4 (最大开仓数)
**借用**: 这些参数可以作为我们策略的基线, 非随手拍

### 2. king-pin (Python)  
**架构**: 多因子融合 (技术30%+资金40%+情绪20%+新闻10%)
**数据源**: Tushare行情 + TrendRadar舆情 35平台 + DeepSeek新闻
**三层风控**: 环境层面→策略层面→交易层面
**借用**: 分层决策架构 + 打分权重分配模式

### 3. ZGNB (Python)
**战法**: BBI+KDJ低位 (少妇), Peak+KDJ (填坑), 放量突破 (TePu)
**特点**: 纯Python, 配置文件驱动, 多线程
**借用**: 战法模块化设计 (每种战法独立配置)

### 4. StockTradebyZ (Python)
**升级战法**: 暴力K(放量启动阳线), 上穿60放量, SuperB1
**特点**: Pipeline架构, 统一过滤条件
**借用**: 当日过滤+知行约束机制

### 5. xman (C++)
**架构**: 多进程分离 (框架/抢板/集合竞价/半路板/卖出)
**功能**: 集合竞价抢板+盘中抢板+半路板
**借用**: 抢板策略参数和方法

### 6. longhubang (Python)
**AI评分**: 买入含金量(30分)+净买入(25分)+卖出压力(20分)+机构共振(15分)+其他(10分)
**借用**: 评分权重分配 + 龙虎榜数据源

## 关键发现: 行业标准参数

| 参数 | daban | 应用 |
|------|-------|------|
| 触发涨幅 | 8% | 留2%缓冲防封死 |
| 秒均涨幅 | 0.125% | 观测窗口6秒 |
| 秒均换手 | 0.005% | 异动监测 |
| 最大持仓 | 4只 | 分散风险 |
| 买入超时 | 3600s | 超时撤单 |

## 启示
1. 打板系统普遍用 C/C++/Go 而非 Python → 执行速度是关键
2. 多进程分离架构是行业标准 (xman: 框架/抢板/竞价/半路板/卖出)
3. AI评分模型 (longhubang) 可以嵌入选股流程
4. 战法模块化设计 (ZGNB) 可以用在 L2 技术形态层
